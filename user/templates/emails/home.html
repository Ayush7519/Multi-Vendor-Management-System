Vendora

Multi-Vendor E-commerce System

Vendora: is a scalable and feature-rich multi-vendor e-commerce platform that enables vendors to effortlessly create, manage, and scale their own digital storefronts within a unified ecosystem. It is designed to empower individual sellers while offering customers a seamless shopping experience across a wide variety of vendors and products.

For Vendors: Vendora provides a dedicated dashboard for managing products, tracking orders, monitoring inventory, and updating shop details. Vendors can upload multiple images, tag products for search optimization, and handle customer interactions.

For Customers: Customers can browse products across all vendors, apply filters and search queries, add items to a cart or wishlist, and place orders with ease.

For Admins: An admin panel enables complete control of users, vendor verification, data monitoring, and platform maintenance.

Key Features

  • User Registration, Login, and Password Management
  • Vendor Verification and Profile Setup
  • Product Listing with Multiple Images and Tags
  • Smart Search, Filter, and Categorization
  • Cart and Wishlist Functionality
  • Order Placement and Tracking
  • Admin Panel with Vendor Oversight
  • Email-based Password Recovery and Verification

Tech Stack

Layer Technology Description
Backend Django REST Framework Powerful API backend with authentication, permissions, and serialization
Frontend Next.js (Planned) Modern React-based UI framework with SSR/SPA capability
Database PostgreSQL Reliable relational database to store structured data
Auth Token-Based (DRF Token or JWT) Secure authentication and session handling
Media Cloud or Local Handles product images and logos

API Endpoints

POST /user/registration/ — Register a new user
POST /user/login/ — Login a user
POST /vendor/verified/<uuid:pk>/ — Verify a vendor account
GET /user/list/ — Retrieve a filtered list of users
GET /single/user-view/<uuid:pk>/ — Retrieve details of a specific user by ID
DELETE /user/delete/<uuid:pk>/ — Delete a user account from the system
GET /login/user/profile/ — Retrieve the profile of the currently logged-in user
GET /vendor/profile/<uuid:pk>/ — Create a vendor profile
UPDATE /user/profile/update/<uuid:pk>/ — Update the user profile
POST /user/password-change/ — Change user password with the old password
